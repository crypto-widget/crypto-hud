## 報告摘要

本報告記錄 Crypto HUD 在下述測試條件中的 CPU、記憶體、執行緒、程序結構、啟動、
網路請求與穩定性數據。

預設設定為 3 個交易對、每 5 秒更新一次。熱快取運行 10 分鐘時，記錄到的平均
整機 CPU 為 **0.070%**，私有提交記憶體中位數為 **20.27 MiB**，工作集中位數為
**47.96 MiB**，私有工作集中位數為 **19.29 MiB**。

在 20 個即時交易對、相同 5 秒更新間隔下，平均 CPU 為 **0.125%**，CPU P95
為 **0.681%**，私有提交記憶體中位數為 **21.56 MiB**，工作集中位數為
**49.02 MiB**。

所有受測情境都沒有建立子程序或 WebView2 程序。系統匣隱藏後的程序記憶體與離線
可見情境接近。10 分鐘測試中的記憶體在預熱與 5 分鐘 K 線更新階段上升，並在
第 7–10 分鐘維持於一個穩定區間。

本報告數值描述指定測試機上的 Crypto HUD 程序，不包含歸因到 GPU、DWM 合成或
實際電功耗的資料。

## 指標說明

- **整機 CPU：**程序 CPU 時間除以實際時間與 12 個邏輯處理器，對應 Windows
  工作管理員常用的整機百分比口徑。
- **CPU P95：**穩態樣本中 95% 的 CPU 數值不高於該值。
- **私有提交：**只配置給該程序的已提交記憶體。
- **工作集：**目前駐留在 RAM 中的記憶體頁，可能包含共用頁。
- **私有工作集：**該程序獨享的駐留 RAM，最接近工作管理員常見的程序記憶體數值。
- **執行緒數：**每個取樣點的程序執行緒總數；暫時網路執行緒會使該值變化。
- **控制代碼數：**程序持有的 Windows 控制代碼數量。

CPU 使用 `TotalProcessorTime` 增量計算。一般穩態情境排除前 10 秒；5 分鐘情境
排除前 30 秒；10 分鐘情境排除第一分鐘。

首次離線彙總存在整數捨入問題，因此離線 CPU 值依單核心等效計數器重新換算為概算值。
即時連線 CPU 值不受該捨入問題影響。

## 情境間觀察差異

- 設定視窗情境的私有提交記憶體中位數為 23.74 MiB，10 分鐘預設情境為
  20.27 MiB；工作集中位數分別為 55.35 MiB 與 47.96 MiB。兩個情境持續時間
  不同，因此此處只陳述觀察值。
- 即時交易對從 3 個增加到 20 個後，平均 CPU 從 0.070% 變為 0.125%，私有提交
  從 20.27 MiB 變為 21.56 MiB，工作集從 47.96 MiB 變為 49.02 MiB。
- 離線情境中，10 個 Quote Board 比 1 個可見小工具多使用 2.70 MiB 私有提交與
  3.55 MiB 工作集。
- 五種小工具各運行一個時，比離線單一小工具情境多使用 9.20 MiB 私有提交與
  11.44 MiB 工作集。
- 300% 縮放比離線預設縮放多使用 1.28 MiB 私有提交與 2.37 MiB 工作集。
- 系統匣隱藏前後私有提交均為 17.43 MiB；工作集從 41.93 MiB 變為 41.77 MiB。
  捨入後的離線 CPU 概算值分別為 0.04% 與 0.03%。

## 啟動測量

即時連線啟動情境在 **1.705 秒**後達到既有的「全部行情資料就緒」標記。

該標記要求已設定的行情列都有資料。時間包含外掛探索、視窗建立、首批行情請求，以及
每秒一次、用於消費行情事件的 UI 計時器。該數值不是首幀或首次顯示視窗時間。

生產式更新檢查在測試環境存取 GitHub Releases API 失敗，因此該情境記錄的是失敗
路徑。程序繼續運行；成功更新回應沒有測量。

## 網路與檔案觀察

穩態請求量近似為：

`唯一交易對數量 × (60 / 更新秒數 + 0.2)` 次/分鐘。

`0.2` 表示每 5 分鐘一次 K 線更新。

| 設定 | 估算請求數/分鐘 |
| --- | ---: |
| 3 個交易對，5 秒更新 | 36.6 |
| 8 個交易對，5 秒更新 | 97.6 |
| 20 個交易對，5 秒更新 | 244 |
| 3 個交易對，60 秒更新 | 3.6 |
| 20 個交易對，60 秒更新 | 24 |

啟動時，每個交易對約增加一次 ticker 請求與一次 K 線請求。網路延遲會延長單一
週期，因此實際請求速率可能低於公式結果。

本機 Proxy 拒絕連線情境在 90 秒內記錄到 4 個失敗行情週期。該情境涵蓋立即拒絕
連線，沒有涵蓋 8 秒讀取逾時或斷線後重新連線。

啟動與預熱後，靜置情境記錄到的檔案讀寫幾乎為零。沒有測量慢速磁碟上的拖曳儲存與
設定儲存延遲。Windows 程序 I/O 計數器沒有用作程序網路位元組統計。

## 測試範圍與限制

- 測試機使用 QEMU 與遠端虛擬顯示，未量化實體 GPU、DWM 合成與電功耗。
- 每個自訂情境執行一次，本報告不包含多輪信賴區間。
- 最長運行時間為 10 分鐘，沒有執行 8–24 小時測試。
- 未測量系統 150–300% DPI、4K 多螢幕、移除螢幕、睡眠喚醒與最短間隔持續動畫。
- 網路故障情境使用 Proxy 立即拒絕連線；未測量 DNS 卡住、慢速回應、HTTP 429、
  8 秒逾時與網路恢復。
- 目前環境無法使用 Windows Performance Recorder 與 WPA 歸因。本報告不包含程序
  網路位元組、GPU 使用率、內容切換、獨立喚醒次數與電功耗。
- 未在慢速或同步磁碟測量拖曳與設定儲存延遲。
- 結果只對應報告頂端所列的 0.9.7 版本與執行檔雜湊。
